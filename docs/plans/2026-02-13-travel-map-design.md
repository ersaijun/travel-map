# 中国5A景区全揽网站 - 设计文档

> 创建日期：2026-02-13

## 一、项目概述

### 1.1 项目背景
中国现有358家国家5A级旅游景区，本项目旨在构建一个整合所有5A景区信息的综合性旅游网站，为用户提供景区查询、详情展示、行程规划等服务。

### 1.2 核心功能
1. **交互式地图展示**：在地图上直观显示所有5A景区的地理位置分布
2. **景区信息展示**：展示景区百科信息、详细介绍、特色亮点、门票信息、开放时间等
3. **省份筛选功能**：按省份分类查询景区
4. **旅游规划功能**：帮助用户创建和管理旅游行程

### 3.2 API设计

#### 景区相关API
```
GET    /api/spots              # 获取景区列表（支持分页、省份筛选）
GET    /api/spots/:id          # 获取景区详情
GET    /api/spots/provinces    # 获取省份列表及景区数量
GET    /api/spots/search       # 搜索景区
```

#### 用户相关API
```
POST   /api/auth/wechat        # 微信登录
POST   /api/auth/qq            # QQ登录
GET    /api/user/profile       # 获取用户信息
PUT    /api/user/profile       # 更新用户信息
```

#### 收藏相关API
```
GET    /api/user/favorites     # 获取收藏列表
POST   /api/user/favorites     # 添加收藏
DELETE /api/user/favorites/:id # 取消收藏
```

#### 行程相关API
```
GET    /api/plans              # 获取用户行程列表
POST   /api/plans              # 创建行程
GET    /api/plans/:id          # 获取行程详情
PUT    /api/plans/:id          # 更新行程
DELETE /api/plans/:id          # 删除行程
POST   /api/plans/generate     # AI生成行程（预留接口）
```

## 四、数据模型设计

### 4.1 景区数据模型 (ScenicSpot)

```typescript
interface ScenicSpot {
  _id: ObjectId;
  name: string;                    // 景区名称
  province: string;                // 所属省份
  city: string;                    // 所属城市
  location: {                      // 地理位置
    type: 'Point',
    coordinates: [number, number]  // [经度, 纬度]
  };
  address: string;                 // 详细地址
  description: string;             // 景区介绍
  highlights: string[];            // 特色亮点
  category: string;                // 景区类型（自然风光、历史古迹、主题公园等）
  ticketInfo: {                    // 门票信息
    price: number;
    description: string;
  };
  openTime: string;                // 开放时间
  rating: number;                  // 评分
  images: string[];                // 图片URL列表
  wikiUrl: string;                 // 百科链接
  createdAt: Date;
  updatedAt: Date;
}
```

### 4.2 用户数据模型 (User)

```typescript
interface User {
  _id: ObjectId;
  openid: string;                  // 微信/QQ openid
  unionid?: string;                // 微信 unionid
  loginType: 'wechat' | 'qq';      // 登录方式
  nickname: string;
  avatar: string;
  favorites: ObjectId[];           // 收藏的景区ID
  createdAt: Date;
  updatedAt: Date;
}
```

### 4.3 行程规划模型 (TravelPlan)

```typescript
interface TravelPlan {
  _id: ObjectId;
  userId: ObjectId;
  title: string;
  startDate: Date;
  endDate: Date;
  spots: [{
    spotId: ObjectId;
    day: number;                   // 第几天
    order: number;                 // 当天顺序
    notes: string;                 // 备注
  }];
  createdAt: Date;
  updatedAt: Date;
}
```

## 六、技术难点与解决方案

| 难点 | 问题描述 | 解决方案 |
|------|----------|----------|
| 358个景区点位性能 | 大量Marker可能导致地图卡顿 | 使用高德地图点聚合功能，缩放时动态聚合/展开点位 |
| 景区坐标数据获取 | 358个景区的精确坐标需要手动整理 | 使用高德地图地理编码API，根据景区名称自动获取坐标，人工校验 |
| 百科信息获取 | 百度百科无官方API | 使用爬虫抓取百科内容（需遵守robots.txt），或使用维基百科API |
| 微信/QQ登录集成 | 需要企业资质申请开放平台账号 | 开发阶段使用测试账号，上线前申请正式资质 |
| 数据更新维护 | 5A景区名单每年更新 | 建立数据管理后台，支持批量导入更新，订阅官方公告 |
